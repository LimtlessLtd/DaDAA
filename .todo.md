# DaDAA Project Plan (.todo.md)


- [x] **Phase 1: The Ears (Audio & Transcription)**
  - [x] Establish Node.js Discord bot voice connection.
  - [x] Route voice streams to the local Python transcription server.
  - [x] Output clean, speaker-labeled text in real time.


- [x] **Phase 2: The Brain (Context & Relevancy)**
  - [x] Build an OOC (Out-Of-Character) filter to ignore table talk and jokes.
  - [x] Integrate local Foundry VTT data (NPCs, items, lore) into the context pipeline.
  - [x] Implement short-term memory Summarization to prevent AI context bloat.
  - [x] Design an "importance filter" so the AI only flags critical triggers.
  - [x] Implement player character log, associate which users in discord control which characters in Foundry VTT and the AI can log the players character development, traumatic experiences, notable NPCs and major plot events in their life.
  - [x] Implement a "next session" feature which will guide The Brain on what next session might look like and allows for planning for the future.


- [ ] **Phase 3: The Voice (In-Character Speech & Prompts)**
  - [x] Set up Text-to-Speech (TTS) so the AI can speak directly back into the Discord channel.
  - [ ] Implement in-character narrative delivery (DM persona, tone, pacing).
  - [ ] Code check-trigger logic (e.g., the AI prompts: "Roll a DC 12 Athletics check").
  - [ ] Build a success/failure outcome system to handle rolls and narrative results.
  - [ ] Implement consequences for success and failure outcomes.
  - [ ] Implement RAG database instead of text files.


- [ ] **Phase 4: The Hand (Actions & Outputs)**
  - [ ] Build a Foundry VTT API bridge to trigger rolls or update game states and cast spells or give items/gold etc.
  - [ ] Implement automated "Session Note" triggers based on FoundryVTT events, such as combat encounters and where player tokens move within FoundryVTT scenes.
  - [ ] Auto generate session content on the fly, including NPCs, their images and backstories as well as journal entries, items, rolltables etc.
